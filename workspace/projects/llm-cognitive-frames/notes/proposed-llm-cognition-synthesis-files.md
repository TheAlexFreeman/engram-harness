## High-Priority Synthesis Opportunities

1. **knowledge/cognitive-science/llm-predictive-processing-active-inference.md**
   - **Thesis**: Next-token prediction as approximate Bayesian inference under a generative model; hallucinations as failures in precision-weighting of priors vs. sensory (prompt) evidence. Engram-style persistent memory acts as an external "generative model" update mechanism, enabling better active inference across sessions.
   - **Practical implications for Engram**: How memory governance (trust levels, review queues) functions as precision control. Design patterns for prompts/interfaces that supply "sensory" evidence to reduce confabulation. Links to dynamical-systems and umwelt files.
   - **Why interesting**: Bridges core cognitive science (Friston, Clark) directly to LLM mechanics and memory system design. Fills gap between existing dynamical-systems file and practical agent behavior.

2. **knowledge/cognitive-science/llm-narrative-scaffolding-cognition.md**
   - **Thesis**: LLMs' strength in narrative generation is not a bug but a feature for organizing semiotic frames (building on umwelten file and existing narrative-cognition.md). Narrative provides coherent "affordance landscapes" and compresses long-term memory. In agents, internalized narrative dialogue (chain-of-thought, self-critique) mimics human narrative self.
   - **Practical for Engram**: Recommendations for structuring memory files as narrative "episodes" or "chapters"; using narrative in SUMMARY.md and cross-references; risks of narrative lock-in (confirmation bias in memory).
   - **Why interesting**: Directly addresses user's core interest in narrative cognition. Practical playbook for how we are already using narrative in this knowledge base.

3. **knowledge/ai/frontier/cognition/llm-confabulation-phenomenology-and-mitigation.md**
   - **Thesis**: Confabulation is the LLM analogue of Dennett's "user illusion" and Hoffman's interface theory — the model generates plausible but ungrounded reports. Memory systems like Engram provide external grounding and verification loops, turning confabulation into a feature for creative exploration when calibrated.
   - **Practical**: Interface design (e.g., this harness's verification, review-queue.md), HITL approvals, trust-frontmatter as phenomenological calibration. Links to interpretability (representation-confabulation.md) and extended-mind files.
   - **Why interesting**: Synthesizes recent consciousness-knowledge-files with frontier interpretability and practical governance.

4. **knowledge/cognitive-science/llm-hemispheric-analogues-and-specialization.md**
   - **Thesis**: Transformer attention patterns mirror McGilchrist's left (narrow, focused, "grasp") vs right (broad, contextual, "presence") hemisphere. MoE architectures and multi-agent systems explicitly implement this division of labor. Implications for agent cognition: avoid "left-hemisphere dominance" in over-optimized tool use.
   - **Practical for Engram**: Design sub-agents or tool policies that balance broad exploration (umwelt expansion) and precise execution. Use in retrieval (broad semantic vs precise keyword).
   - **Why interesting**: Synthesizes the brand-new McGilchrist bridge files with existing architectures (MoE, agent patterns) and umwelt/affordance work. Highly original.

5. **knowledge/ai/frontier/cognition/engram-as-cognitive-scaffold-synthesis.md** (or promote from working note)
   - **Thesis**: Comprehensive synthesis: Engram realizes extended mind + active inference + narrative scaffolding + governed cultural evolution for LLMs. Practical implications section as a checklist/playbook for designing interfaces, memory taxonomy, governance, and evaluation that augment rather than merely query LLM cognition.
   - **Why interesting**: Meta-file that ties the entire llm-cognitive-frames project together, with direct applicability to improving this very system. Could update frontier-synthesis.md.
