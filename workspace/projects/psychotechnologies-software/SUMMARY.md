## Resolved questions (2)

1. ~What historical or theoretical psychotechnologies (literacy, diagramming, etc.) parallel modern ones?~
   → AI coding agents shift programming from low-level symbol manipulation and high working-memory load (syntax, boilerplate, search) to higher-order orchestration, verification, system-level design, and intention articulation. Developers practice more meta-cognition (specifying success criteria clearly, knowing when to intervene), relevance realization (agents surface connections in large contexts), and declarative thinking ("what should exist" vs "how to code it"). This mirrors extended mind (agents as cognitive prosthetics) and can amplify 4E aspects — enactive interaction with the agent "environment." However, it risks deskilling in fundamentals or over-reliance if not paired with deliberate verification practices. In practice (per recent sessions and landscape), it accelerates prototyping but requires strong oversight for production code. (2026-04-27)
2. ~What specific new tools/practices from 2024-2026 qualify as psychotechnologies for software devs?~
   → Specific new tools/practices (2024-2026): AI-augmented IDEs like Cursor and GitHub Copilot Workspace for real-time collaborative editing and codebase-aware suggestions; autonomous agents like Devin (Cognition), Claude Code, and advanced Codex/Gemini tools that handle planning, execution, testing, and PRs; advanced prompt engineering as a deliberate cognitive discipline; multi-agent workflows for distributed task orchestration; meta-practices for collecting and internalizing cognitive techniques tailored to dev (e.g., debugging mindsets, flow triggers). These qualify as psychotechnologies because they standardize and extend cognitive processes like relevance detection, intention structuring, verification, and distributed problem-solving. (2026-04-27)
