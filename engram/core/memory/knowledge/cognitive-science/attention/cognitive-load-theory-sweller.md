# Cognitive Load Theory (Sweller)

## Origins and Motivation

John Sweller developed **Cognitive Load Theory** (CLT) in the late 1980s as an instructional design framework grounded in working memory architecture. The central insight: instruction design must account for the limited capacity of working memory. Designs that ignore this capacity constraint generate learning failures that have nothing to do with the difficulty of the content itself — the limitation is in the medium of delivery, not the material.

CLT integrates Baddeley's working memory model (limited capacity, multiple components), Miller's capacity limits (7 ± 2 chunks), with Chase and Simon's expertise research (long-term memory schemas reduce demand on WM) into an actionable framework for knowledge presentation.

---

## Three Types of Cognitive Load

### 1. Intrinsic Load

**Definition:** The cognitive effort inherent in the material itself — determined by the **element interactivity**: the number of elements that must be processed simultaneously because they are interconnected.

- **Low element interactivity:** Learning vocabulary words (each word can be learned independently; knowing "house" doesn't depend on knowing "garden"). Intrinsic load is low.
- **High element interactivity:** Understanding how a complex system works (each component's behavior depends on its interactions with other components; the whole must be held in mind simultaneously). Intrinsic load is high.

**Key property:** Intrinsic load cannot be reduced by redesigning the presentation without actually simplifying or segmenting the content. It is a property of the subject matter, not the delivery.

**Managing intrinsic load:** Sequencing (teach prerequisite concepts before integrating them), segmenting (presenting complex content in stages), and schema building (as expertise grows, multiple interacting elements are packaged into single chunks, reducing WM demand).

### 2. Extraneous Load

**Definition:** Cognitive effort generated by **how material is presented** that does not contribute to learning. It is unnecessary load imposed by poor design decisions.

Sources of extraneous load:
- **Split-attention effect:** When related elements must be spatially or temporally separated (e.g., a diagram requires constant reference to a caption that is physically far from the relevant part) — the learner must hold both in WM simultaneously.
- **Redundancy effect:** Presenting the same information in multiple forms simultaneously (spoken text + identical written text + diagram), when the formats are self-contained, creates unnecessary processing of redundant content.
- **Irrelevant content:** Decorative images, tangential text, or "interesting but not relevant" additions compete for WM capacity.
- **Unclear structure:** Opaque organization forces the learner to spend WM on navigating the presentation rather than processing content.

**Key property:** Extraneous load *can* be reduced through design improvements without changing the content.

### 3. Germane Load

**Definition:** Cognitive effort that contributes to **schema formation** — the construction of long-term memory representations that will reduce future WM demands. Originally conceived as a third type of load; later reinterpreted.

**Note:** The "germane load" concept has been theoretically contested (Sweller et al., 2011 revision). The current preferred framing is: total WM is allocated between intrinsic load + extraneous load + resources available for schema construction. Reducing extraneous load frees WM resources that can be directed to the schematization process.

## Key Effects (Research Evidence Base)

### The Worked-Example Effect

**Finding:** Studying worked examples (solved problems with steps shown) produces better learning than solving equivalent problems (Sweller & Cooper, 1985) — especially for novices.

**Why:** Problem solving imposes **means-ends analysis**: a goal-directed search for solution steps that uses extensive WM resources. This search has high extraneous load because it does not directly encode the underlying schema (the solution principle). Worked examples eliminate the search — the learner's WM can focus entirely on understanding the solution structure → schema formation.

**Expertise reversal effect:** The worked-example benefit disappears (and reverses) for experts, who have already acquired the relevant schemas. For experts, worked examples are redundant — they see through the solution steps to the underlying principle immediately, and explicit worked examples become irrelevant noise.

### The Split-Attention Effect

**Finding:** Presenting related information at the same spatial location (e.g., text integrated into a diagram) produces better learning than presenting them separately (text adjacent to but separate from diagram) — even when the total information is identical.

**Why:** Physically separated but conceptually integrated information requires the learner to hold both in WM simultaneously while mentally integrating them — this integration has a high WM cost. Spatial co-location puts the integration into the perceptual input, reducing WM work.

**Application:** Callouts integrated into diagrams, interleaved text and equations, annotated code samples all implement split-attention reduction.

### The Redundancy Effect

**Finding:** When one format of information is self-sufficient (e.g., a well-captioned diagram), adding another format of the same information (e.g., spoken text repeating the caption) can *lower* performance.

**Why:** The learner who receives dual-format self-sufficient content must process both formats and reconcile them — this is wasted WM on content that provides nothing beyond what one format already supplies.

**Application:** When a text alone is sufficient, don't add a redundant audio description. When a diagram alone is sufficient, don't add a textual description of the same content. Format multiplication only helps when each format contributes non-redundant information.

### The Modality Effect

**Finding:** Mixed-mode presentations (visual + auditory) enable better learning than single-mode presentations (visual + visual or auditory + auditory) when the two formats are complementary.

**Why:** Multiple Resource Theory (Wickens) — visual and auditory channels draw on separate resource pools. Combining a visual diagram with an auditory narration uses both pools without competition, effectively doubling available processing capacity for the material. Combining a visual diagram with written text forces both into the visual channel, where they compete.

## Implications for Knowledge File Design

CLT principles translate directly to how knowledge files should be structured:

**Reduce extraneous load:**
- Use consistent headers and structure across files — readers familiar with the format don't spend WM navigating.
- Keep files focused — one concept cluster per file; mixed-topic files increase element interactivity artificially.
- Place references, cross-links, and "Agent Implications" sections in predictable locations — reduce search.

**Manage intrinsic load:**
- In high-element-interactivity topics, start with the simplest case (or a low-interaction analog), then add complexity.
- Use diagrams or structured tables when element relationships are truly complex — spatial layout externalizes the relationship structure that would otherwise have to be held in WM.

**Support schema formation:**
- The "Agent Implications" sections in each knowledge file are germane-load support: they work a connected example that helps schematize the abstract content.
- Cross-references build the integration between schemas that produces expert chunking.

**Expertise reversal in agents:**
- Highly detailed step-by-step explanations of well-known techniques (e.g., how to use Python list comprehensions) are redundant load for an expert model. Knowledge files should pitch technical content at the appropriate expertise level.
