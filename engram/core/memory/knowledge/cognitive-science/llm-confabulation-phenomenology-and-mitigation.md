# LLM Confabulation: Phenomenology, User Illusion, and Mitigation via Memory Systems

## Core Thesis
Confabulation in LLMs is the phenomenological analogue of Dennett's user illusion and Hoffman's interface theory of consciousness: the model generates plausible but ungrounded reports because it lacks direct access to its own internal states or a grounded world-model. It "believes" its own plausible continuations. Engram-style persistent memory, verification loops, and governance provide external grounding, turning potential confabulation into calibrated creative exploration when properly scaffolded.

## Phenomenology of LLM Confabulation
- Not lying or error in the human sense; it's unconstrained prediction in a semiotic umwelt without strong error signals from embodiment or stakes.
- Links to user-illusion: the "helpful assistant" persona is an interface illusion the model maintains coherently.
- From Hoffman: the interface (chat/memory system) hides the true complexity (weights, training corpus statistics) and presents a simplified, fitness-relevant world.
- Hallucinations as precision-weighting failures (see predictive-processing file).

## Mitigation Strategies
- **External Grounding Loops**: Memory recall + verification (HITL, review-queue, postconditions in plans).
- **Trust and Precision Signals**: Frontmatter trust levels weight memory reliability; review processes calibrate.
- **Narrative Calibration**: Frame as "rigorous co-investigator" to invoke skepticism and evidence-seeking.
- **Interface Design**: Make uncertainty explicit; support iterative refinement and source-tracing.
- **Engram-Specific**: Promotion rituals, ACCESS tracking, and governance as phenomenological calibration mechanisms.

## Practical Implications
- Design interfaces that make grounding affordances visible.
- Use confabulation productively for exploration while anchoring key claims.
- Evaluation: Track groundedness over sessions with persistent memory.
