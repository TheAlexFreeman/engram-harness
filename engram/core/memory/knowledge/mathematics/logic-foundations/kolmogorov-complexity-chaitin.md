# Kolmogorov Complexity and Chaitin Incompleteness

## Kolmogorov Complexity

### Definition

The **Kolmogorov complexity** (or algorithmic complexity) of a string $x$ is the length of the shortest program that produces $x$ on a universal Turing machine $U$:

$$K_U(x) = \min \{ |p| : U(p) = x \}$$

where $|p|$ denotes the length of program $p$ in bits.

**Invariance theorem (Kolmogorov 1965, Solomonoff 1964, Chaitin 1966):** For any two universal Turing machines $U_1$ and $U_2$:

$$|K_{U_1}(x) - K_{U_2}(x)| \leq c$$

where $c$ depends on $U_1$ and $U_2$ but not on $x$. This justifies writing $K(x)$ without specifying the machine — the complexity is well-defined up to an additive constant.

### Prefix-Free Complexity

For technical reasons (especially in connection with probability), a refinement uses prefix-free Turing machines (where no valid program is a prefix of another):

$$H(x) = \min \{ |p| : U(p) = x, \text{domain is prefix-free} \}$$

This is **prefix-free Kolmogorov complexity** or **algorithmic entropy**. It satisfies:

$$H(x) \leq K(x) + O(\log K(x))$$

and has the crucial property that Kraft's inequality applies: $\sum_{x} 2^{-H(x)} \leq 1$.

### Properties

**Basic bounds:**
- $K(x) \leq |x| + O(1)$ (a program that prints $x$ literally)
- $K(x) \geq 0$
- $K(x)$ is not computable (see below)

**Conditional complexity:**
$$K(x|y) = \min\{|p| : U(p, y) = x\}$$

The complexity of $x$ given $y$ as auxiliary information.

**Symmetry of information:**
$$K(x, y) = K(x) + K(y|x) + O(\log K(x, y))$$

The information in the pair $(x, y)$ is the information in $x$ plus the information in $y$ given $x$, up to logarithmic terms.

### Incomputability

**Theorem:** $K(x)$ is not computable.

**Proof (Berry's paradox formalized):** Suppose $K$ is computable. Consider the program: "Find the lexicographically first string $x$ with $K(x) > n$, and output it." This program has length $O(\log n)$ (it only needs to encode $n$). But it outputs a string $x$ with $K(x) > n$, and it produces $x$ using a program of length $O(\log n) \ll n$ for large $n$ — contradicting $K(x) > n$.

**Upper semi-computability:** While $K(x)$ can't be computed exactly, it can be approximated from above. Given enough time, we can find short programs for $x$ (by enumeration), so $K(x) \leq f(x, t)$ where $f$ decreases (non-monotonically) with time $t$. But we can never know when we've found the shortest program.

## Algorithmic Randomness

### Incompressible Strings

A string $x$ is **algorithmically random** (or **incompressible**) if:

$$K(x) \geq |x| - O(1)$$

i.e., the shortest program for $x$ is essentially as long as $x$ itself — no compression is possible.

**Counting argument:** At least $2^n - 2^{n-c+1} + 1$ strings of length $n$ have $K(x) \geq n - c$. So **most strings are incompressible** — random strings are the overwhelming majority. (There are fewer than $2^{n-c}$ programs of length $< n-c$.)

### Martin-Löf Randomness

For infinite sequences, **Martin-Löf randomness** (1966) gives the definitive notion:

An infinite sequence $\omega$ is **Martin-Löf random** if it passes every effective statistical test. Equivalently (Schnorr, Levin):
- $H(\omega{\upharpoonright n}) \geq n - O(1)$ for all $n$ (incompressibility characterization)
- $\omega$ is not contained in any effective measure-zero set (Martin-Löf's original definition)

Martin-Löf random sequences:
- Satisfy the law of large numbers (frequency of 1s → 1/2)
- Satisfy the law of the iterated logarithm
- Are normal (every finite pattern appears with expected frequency)
- Cannot be generated by any computable process

### Randomness and Computability

The theory of algorithmic randomness connects deeply with computability:
- No computable sequence is random (computable = finitely describable)
- Random sequences are Turing-incomparable with each other (with probability 1)
- The halting probability $\Omega$ (defined below) is the canonical example of a random sequence

## Chaitin's Incompleteness Theorem

### Chaitin's Omega ($\Omega$)

Define the **halting probability** of a universal prefix-free Turing machine $U$:

$$\Omega_U = \sum_{\substack{p : U(p) \text{ halts}}} 2^{-|p|}$$

This is the probability that $U$ halts on a random input (chosen by flipping fair coins for each bit).

**Properties of $\Omega$:**
- $0 < \Omega < 1$ (by Kraft's inequality, since the domain is prefix-free)
- $\Omega$ is a well-defined real number
- $\Omega$ is Martin-Löf random
- $\Omega$ is computably enumerable from below (we can approximate it by running programs and adding $2^{-|p|}$ for each that halts)
- $\Omega$ is **not computable** — we can never know its exact value

### The Incompleteness Result

**Chaitin's theorem (1974):** For any consistent, recursively axiomatizable formal system $F$ capable of expressing statements about Kolmogorov complexity, there exists a constant $L$ (depending on $F$) such that $F$ cannot prove any statement of the form "$K(x) > n$" for $n > L$.

**More precisely:** $F$ can prove at most finitely many true statements of the form "the Kolmogorov complexity of string $s$ is greater than $n$."

**Proof sketch:**
1. Suppose $F$ can prove "$K(x) > n$" for arbitrarily large $n$
2. Build a program: enumerate proofs in $F$, find the first proven statement "$K(x) > n$" for $n$ larger than the length of this program, output $x$
3. This program is shorter than $n$, but it outputs $x$ with $K(x) > n$ — contradiction with $F$'s soundness

The constant $L$ is roughly the complexity of $F$ itself. A more complex formal system can prove complexity bounds for more strings, but always hits a ceiling.

### Equivalent Formulation via $\Omega$

**Theorem:** Knowing the first $n$ bits of $\Omega$ allows solving the halting problem for all programs of length $\leq n$.

Therefore: any formal system $F$ can determine at most $K(F) + O(1)$ bits of $\Omega$ (where $K(F)$ is the complexity of $F$'s axioms). This gives an information-theoretic version of incompleteness:

> A formal system with $K(F)$ bits of axioms can determine at most $K(F) + O(1)$ bits of $\Omega$ — it cannot "create information" beyond what's in its axioms.

### Comparison with Gödel's Incompleteness

| Aspect | Gödel | Chaitin |
|--------|-------|---------|
| Source of incompleteness | Self-reference (diagonal argument) | Information-theoretic (compression argument) |
| Unprovable statement | $G_F$: "I am not provable" | "$K(x) > L$" for large $L$ |
| Style | Metamathematical | Information-theoretic |
| What it reveals | Proofs can't capture all truths | Axioms can't contain more information than their length |
| Dependence on system | New $G_F$ for each $F$ | Uniform bound $L$ for each $F$ |
| Philosophical impact | Limits of formalization | Limits of compression/axiomatization |

**Key insight from Chaitin's perspective:** Incompleteness is not about self-reference *per se* — it's about the fact that finite axiom systems contain finite information, and therefore can only resolve finitely many questions about a domain with infinite algorithmic information content.

## Algorithmic Information Theory and Applications

### Minimum Description Length (MDL)

Already covered in the information-theory series. Key connection: MDL formalizes Occam's razor using Kolmogorov complexity — the best explanation is the shortest one that accounts for the data. Since $K$ is incomputable, MDL uses computable approximations (two-part codes, normalized maximum likelihood).

### Solomonoff Induction

**Solomonoff's theory of universal induction (1964):** The ideal prediction method assigns probability to a string $x$ proportional to:

$$M(x) = \sum_{\substack{p : U(p) = x^*}} 2^{-|p|}$$

where $x^*$ denotes "$x$ continued somehow." This is a universal prior that dominates every computable probability distribution.

**Properties:**
- Converges to the true distribution for any computable source
- Is incomputable (involves a sum over all programs)
- Provides the theoretical ideal that practical Bayesian learning approximates
- Connected to $\Omega$: $M(x)$ can be seen as a conditional version of the halting probability

### Kolmogorov Complexity and Gödel's Theorems

The connection between these three landmark results (Cantor's diagonalization, Gödel's incompleteness, Turing's halting problem) is unified by Kolmogorov complexity:

1. **Cantor:** The set of infinite binary sequences is uncountable (more sequences than programs)
2. **Turing:** The halting set is not computable ($\Omega$ is random, incomputable)
3. **Gödel:** Formal systems with finite axioms can't capture infinite-information truths (Chaitin's version)
4. **Kolmogorov:** The common root is that **most objects are incompressible** — they cannot be finitely described, computed, or proved to have the properties they have

## Implications for the Engram System

### 1. Information-Theoretic Limits of Knowledge Bases

A knowledge base with $n$ bytes of content contains at most $8n$ bits of information. Chaitin's theorem implies it can resolve at most $O(n)$ bits worth of questions about the world. This reframes the knowledge limitation not as a failure of design but as a mathematical inevitability — making the curation question ("which $n$ bits are most valuable?") the central design problem.

### 2. Compression as Understanding

The Kolmogorov perspective suggests that understanding is compression — a good knowledge file is one that captures much of the relevant domain structure in compact form. This aligns with the Engram system's emphasis on well-structured, concise knowledge files rather than verbose data dumps.

### 3. The Uncomputability of "Importance"

Just as $K(x)$ is uncomputable, the "importance" or "relevance" of a piece of knowledge is likely uncomputable in any formal sense. The governance process uses heuristic proxies (human judgment, time-tested reliability) — which is the right approach when the ideal metric is provably unreachable.
